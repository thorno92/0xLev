Leverage API

Solana leverage positions — REST API with JWT token

Base URL: https://levapi.0xleverage.api — Version 1.0.0 · © 0xleverage.io. All rights reserved.

Setup order

  1. Call GET /health to confirm the server is up.
  2. Call POST /check_wallet with apikey and user_id (wallet address) to get a JWT token and trade wallet.
  3. Send the JWT token in the Authorization header: Authorization: <jwt_token>
  4. All POST endpoints (except /check_wallet) require both a valid JWT token (in the header) and a valid apikey in the body (so random clients cannot call the API).
  5. Use the other endpoints in any order (quote → sol_balance → wlcheck → open → positions → close, etc.).

  })
});

Error codes

All errors return JSON: {"error": "message"}. On 401, get a new JWT token via POST /check_wallet.

CodeMeaningExample / when
200OKSuccess
201CreatedWhitelist request submitted (wlrequest)
400Bad RequestMissing/invalid body, apikey, contract_address, trade_id; contract not whitelisted; position size exceeded; already open on token; insufficient balance; invalid token_amount (1–100); leverage/amount not numbers; etc.
401UnauthorizedMissing or invalid JWT token; invalid apikey
403ForbiddenTrade not found or access denied (e.g. close another user’s trade)
404Not FoundUser not registered; trade not found; wallet not found; soltrack no data
406Not AcceptableSelling Error (close failed)
422UnprocessableCouldnt sell (close attempted but failed)
429Too Many RequestsRate limit exceeded; retry after a short delay
500Server ErrorInternal server error; DB/whitelist/TP-SL error
502Bad GatewayFailed to fetch balance (sol_balance)
503UnavailableUnable to fetch user wallets/balance; user wallet creation failed; try again later

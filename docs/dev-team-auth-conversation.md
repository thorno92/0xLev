# Dev Team Auth Flow Conversation (March 16, 2026)

Context from the dev team (Looper, Sir Tris, TuOh3) about how the 0xLeverage API authentication works.

## Key Takeaways

1. **API Key** is shared across all users. It identifies the dapp/protocol, not individual users.
2. **JWT** is per-user, generated by the first handshake (`check_wallet`).
3. JWT is valid for **12 hours** (can be extended if needed).
4. JWT is an encrypted `user_id` used to validate who is making the request.

## Auth Flow (as described by TuOh3)

1. Connect Wallet
2. Sign transaction to ensure wallet ownership
3. API Key is universal (same for all users, identifies the protocol)
4. JWT token is generated by the first handshake of the user
5. JWT is valid for 12 hours
6. The JWT is basically an encrypted user_id so they can validate who is making the request

## Implementation Status

- API key is stored server-side as `OXL_API_KEY` env var
- JWT caching is implemented in `server/leverage-api.ts` (in-memory Map with 12h expiry)
- `check_wallet` endpoint is wired through `trpc.leverage.connectWallet`
- Solana wallet adapter packages are installed but the full UI integration (Phantom/Solflare modal, signature verification) needs to be built
